AIDE : Quoi que c'est NSLookUp ???

 

1 - Démarrer NSLookUp : Où que c'est qu'on met la clé ?

    Pour démarrer NSLookUp, il suffit de lancer le fichier nslookup.jar :

  •     Sous Windows : double-cliquer sur le fichier qui s'exécutera à l'aide de la JVM qui devra être préalablement installée.
  •     Sous Linux : lancer la commande suivante dans le répertoire du jar : java -jar nslookup.jar

    La fenêtre de l'application devrait alors apparaître.

 

2 - Se connecter à un Name Service : Allo ?

    Pour se connecter à un Name Service (NS), cliquer sur Ajouter un NS (dans le menu CORBA, la barre d'outil ou le menu contextuel), une fenêtre de dialogue vous demandera alors l'adresse IP (ou nom résolvable par DNS) et le port sur lequel le NS est lancé. Si l'une ou l'autre des informations saisies n'est pas correcte alors un message d'erreur le signalera.

    Une fois connecté, le NS apparaîtra dans l'arborescence avec tous ses Naming Contexts et CORBA Object. Il est possible de se connecter à autant de NS que souhaité sur le réseau.

 

3 - Les objets manipulés : Ca trifouille ! Ca trifouille !!!

    NSLookUp vous permet de manipuler les Name Services et tous les objets qu'ils contiennent. Il est possible de se dé/connecter à un NS, d'Ajouter ou de retirer un Naming Context et de le déplacer par un glisser déposer dans l'arbre sous condition de le faire lier un Name Service ou à un Name Context. De même, il est possible de dé/lier un Object CORBA. et de le déplacer.

    Les opérations d'ajout et de retrait se font à l'aide du menu, de la barre d'outil ou du menu contextuel accessible par un clique droit. Les déplacement se font par des glisser -déposer. Un bouton supplémentaire vous permet de rafraîchir l'arborescence. Ce rafraîchissement se fait automatiquement toute les 30 secondes.

 

4 - Propiétés : Quoi qu'il y a là-dedans ?

    Pour tous les objets sauf la racine (réseau) il est possible de demander les propriétés, on y retrouve en commun :

  • Le nom déclaré dans le context : utilisable pour une résolution de nom dans un Naming Context.
  • Le type d'object (Name Service, Naming Context ou CORBA Object).
  • Le type IDL : indiquant l'interface IDL implantée par l'objet lié.
  • Le chemin dans l'arborescence des NC : utilisable pour naviguer dans un Name Service.
  • L'IOR : utilisable pour une résolution d'IOR dans un ORB.

Les propiétés d'un NS montre en plus l'adresse et le port de celui-ci.

 

5 - Sauvegarder et charger une configuration : J'vous dépose ça où ?

    Il est possible de sauvegarder la configuration de NSLookUp, c'est à dire de sauvegarder les coordonnées des Name Services auquel on est connecté. Ces informations seront enregistrées dans un fichier .cfg. Lors du chargement d'une configuration, si des NS ne sont pas joignable des erreurs seront remontées.

    Les actions de sauvegarde et de chargement se font en appuyant sur les boutons sauvegarder/charger une configuration du menu fichier ou de la barre d'outil.

 

6 - Quitter : Runawwwwaaaayyy !!!!

    Pour quitter l'application, vous pouvez soit appuyer sur la croix en haut à droite, soit sur le bouton Quitter du menu fichier ou soit faire le raccourci clavier de de fermeture de fenêtre de votre OS.
